# 服务器和构建工具

## 1. Webpack 开发服务器命令

### 完整命令示例
```bash
node --max-old-space-size=16384 ./node_modules/webpack/bin/webpack.js serve --mode development --progress --config web/webpack.config.ts
```

### 参数作用拆解

#### serve
- 调用的是 webpack-dev-server（或 webpack 内置的 dev server）
- 这个模式会启动一个本地开发服务器，保持进程不退出
- 自动监听文件变化，触发重新编译
- 如果配置了 hot: true，就会启用 Hot Module Replacement (HMR)

#### --mode development
- 决定编译优化策略（保留 source map、不压缩）
- 本身不负责持续运行，只是告诉 webpack 用开发模式

#### 其他参数
- --progress → 显示编译进度
- --config web/webpack.config.ts → 指定配置文件
- --max-old-space-size=16384 → 给 Node.js 分配更大的内存

### 总结
持续运行、监听文件变化、支持 HMR 的标识就是 serve。是否启用 HMR 还要看你的 webpack.config.ts 里是否配置了 devServer.hot = true。

## 2. 构建命令对比

### web:build:prod
- 用 webpack 打包并产出可直接运行的前端静态资源（bundle、chunk、CSS、assets）

### build:packages
- 用 tsc --build 只做 TypeScript 的编译/类型检查与增量构建
- 不做打包或代码分割

## 3. Hot Module Replacement (HMR)

HMR 是 webpack 提供的一种功能，允许在应用程序运行时替换、添加或删除模块，而无需完全刷新页面。这使得开发者可以在不丢失应用程序状态的情况下实时查看代码更改的效果。

## 4. React 热重载

### 特点
- React 热重载不会刷新 ts 文件，只会刷新 tsx 文件
- React 使用 JSX/TSX，它本质上是 JavaScript 表达式，不是模板
- JSX 在编译阶段只是转成 React.createElement(...) 调用

## 5. 服务器相关配置要点

### 内存配置
- `--max-old-space-size=16384`：分配 16GB 内存给 Node.js 进程，防止在大型项目构建时出现内存不足错误

### 开发模式配置
- `--mode development`：启用开发模式，保留 source map，不进行代码压缩

### 配置文件指定
- `--config web/webpack.config.ts`：指定 webpack 配置文件路径

## 6. 常见服务器命令模式

### 开发服务器
```bash
# 启动开发服务器
webpack serve --mode development --config webpack.config.js
```

### 生产构建
```bash
# 生产环境构建
webpack --mode production --config webpack.config.js
```

### 监听模式
```bash
# 监听文件变化并重新构建
webpack --watch --mode development
```

### 进度显示
```bash
# 显示构建进度
webpack --progress --mode development
```

## 7. npx 服务器工具

### npx 是什么
npx 是一个随 npm 一起提供的包执行工具，允许用户直接运行 Node.js 包中的命令，而无需全局安装。

### 安装轻量服务器
```bash
yarn add serve
```

### 启动服务器
```bash
npx serve /home/shihao/repo/Work/frontend/Eastlake_Planning/web/.webpack/
```

**注意：** 传入的目录里需要有 index.html，这个网页程序入口

### 修改端口
```bash
npx serve -p 7788
```

## 8. 服务器工具对比

### 1. webpack-dev-server
**定位：** Webpack 官方的开发服务器。

**作用：**
- 自动调用 webpack 打包并在内存中提供构建结果（不会写入磁盘）
- 支持热更新 (HMR)，代码改动后浏览器自动刷新或局部替换
- 提供本地开发环境，通常跑在 http://localhost:8080

**特点：**
- 深度集成 webpack 配置
- 适合前端开发阶段，不适合直接部署到生产环境

### 2. npx serve（通常指 npx serve）
**定位：** 一个轻量级的静态文件服务器（来自 vercel/serve）。

**作用：**
- 把某个目录（比如 dist/）里的静态文件直接通过 HTTP 提供访问
- 常用于预览打包结果或快速部署静态站点

**特点：**
- 不会打包或编译，只是简单地提供文件
- 没有 HMR，没有 webpack 集成
- 适合生产环境下的静态资源托管

### 3. http-server
**定位：** Node.js 社区的经典静态文件服务器（http-server）。

**作用：**
- 类似 serve，把某个目录直接通过 HTTP 提供访问
- 常用于本地测试或轻量级静态站点部署

**特点：**
- 功能简单，配置少
- 没有打包、没有 HMR
- 更偏向于"临时预览"或"快速搭建一个静态文件服务"

### 工具对比表

| 特性 | http-server | npx serve (Vercel) |
|------|-------------|-------------------|
| 定位 | 经典的 Node.js 静态文件服务器 | Vercel 官方的现代静态站点预览工具 |
| 维护情况 | 历史悠久，更新频率较低 | 活跃维护，近几年仍在频繁更新 |
| 配置能力 | 几乎零配置，支持 HTTPS | 支持 serve.json、package.json 等多层配置，灵活可定制 |
| 功能 | 提供目录和文件访问，支持 HTTPS | 支持 SPA（单页应用）路由重写、目录列表、跨域配置、SSL 等 |
| 使用场景 | 本地快速预览静态文件 | 本地开发、团队协作、临时部署、生产环境静态托管 |
| 默认体验 | 简单直接，命令行启动 | 零配置启动，默认端口 3000，体验更贴近现代前端工作流 |

### http-server vs npx serve
总体来说，npx serve（Vercel 出品）比 http-server 更现代化，功能更丰富，维护更活跃；而 http-server 更偏向传统、轻量的零配置静态文件服务器。

## 9. Webpack 打包完成后的使用

### 打包结果
打包完成之后，Webpack 会在你配置的 output 目录（通常是 dist/ 或者 build/）生成一套静态资源文件（HTML、JS、CSS、图片等）。这些文件就是你要上线部署的前端应用。

### 使用方式

#### 1. 本地预览
更推荐用一个静态服务器来预览，比如：
```bash
npx serve dist
```

或者用 http-server：
```bash
npx http-server dist
```

**注意：** dist 下需要有 index.html

#### 2. 部署到 Web 服务器

##### Nginx / Apache
把打包后的文件放到服务器的根目录（如 /var/www/html），配置好静态资源路径即可。

##### Node.js + Express
用 express.static 中间件托管 dist/ 目录。

## 10. Webpack 命令详解

### 调用 Webpack 打包工具
```bash
webpack
```

### 参数说明

#### --mode production
指定构建模式为生产环境。Webpack 会自动启用优化（压缩、tree-shaking 等）。

#### --progress
在构建过程中显示进度条，方便观察打包进度。

#### --config web/webpack.config.ts
指定配置文件路径，这里用的是 TypeScript 写的 Webpack 配置文件（webpack.config.ts），位于 web 目录下。

### 执行效果
运行这个脚本时，Webpack 会：
1. 使用 web/webpack.config.ts 作为配置
2. 按照生产模式打包前端代码
3. 输出优化后的静态资源（通常在 dist/ 或配置指定的目录）
4. 在终端显示打包进度

## 12. Electron

### 定义
Electron 是一个用 JavaScript、HTML、CSS 构建桌面应用的框架。

### 作用
- 内置 Chromium（浏览器内核）和 Node.js
- 让你用前端技术写桌面应用，跨平台运行在 Windows、macOS、Linux

### 特点
- 一个代码库即可跨平台
- 可以调用操作系统 API（文件系统、菜单、通知等）
- 常见应用：VSCode、Slack、Discord 都是用 Electron 构建的

### Webpack vs Electron

| 特性 | Webpack | Electron |
|------|---------|----------|
| 类型 | 前端构建工具 | 桌面应用框架 |
| 主要用途 | 打包 JS/CSS/资源供浏览器使用 | 用 Web 技术写桌面应用 |
| 技术核心 | Loader + Plugin | Chromium + Node.js |
| 输出结果 | 浏览器可运行的 bundle.js | 可执行的桌面应用程序 |
| 典型场景 | React/Vue 项目打包 | VSCode、Slack、Discord 等桌面软件 |

### 总结
Webpack 是前端项目的打包工具，解决"如何把代码和资源打包到浏览器"；Electron 是桌面应用框架，解决"如何用前端技术写桌面软件"。两者经常结合使用：先用 Webpack 打包前端代码，再用 Electron 封装成桌面应用。
