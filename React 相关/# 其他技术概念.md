# 其他技术概念

## 1. 浏览器渲染机制

### JS 线程和 UI 线程为什么互斥
浏览器内部有多个线程：
- **JS 线程：** 负责执行 JavaScript 代码
- **UI 渲染线程：** 负责布局（reflow）和绘制（repaint）

这两个线程是互斥的，因为 JS 可以直接操作 DOM。如果在 JS 修改 DOM 的同时 UI 线程也在渲染，就可能导致不一致甚至崩溃。所以浏览器规定：当 JS 在运行时，UI 渲染线程必须暂停。

### 当 JS 影响 UI 时会发生什么

#### 重排（Reflow）
当 JS 改变了 DOM 结构或元素的几何属性（大小、位置），浏览器需要重新计算布局。

#### 重绘（Repaint）
当 JS 改变了元素的样式（颜色、背景等），浏览器需要重新绘制像素。

这些操作都会触发 UI 线程工作，而 JS 线程必须先让出执行权。

### 性能影响
- 如果 JS 执行时间过长，UI 渲染线程就一直被挂起 → 页面卡顿
- 如果频繁触发重排/重绘（比如在循环里不断修改 DOM），就会导致掉帧

这也是 React 引入 Concurrent Mode 的原因之一：通过任务拆分和优先级调度，避免长时间阻塞 UI。

### 总结
- JS 线程和 UI 线程互斥，是为了保证 DOM 操作和渲染的一致性
- 当 JS 修改 DOM 或样式时，会触发重排（Reflow）和重绘（Repaint）
- 如果 JS 执行过久或频繁触发这些操作，就会造成页面卡顿

## 2. React Hooks 使用规则

### 仅在顶层使用 Hooks
不要在循环、条件或嵌套函数中调用 Hook，确保 Hooks 在每次渲染时都以相同的顺序被调用。

### 使用 ESLint 插件
React 官方提供了 eslint-plugin-react-hooks 插件来帮助你检查 Hook 的使用是否正确。非命令行，需要在配置文件里配置。

### 不能在不确定是否稳定渲染生成的嵌套组件里使用 React 钩子
当 React 的组件是嵌套的，且被嵌套的子组件通过 if/else 判断，不确定能否稳定会被渲染生成，且子组件里使用了钩子函数。

当组件刷新时，子组件需要生成，会调用钩子函数，导致重新渲染，渲染时又会调用钩子函数，导致钩子数量不一致，引发报错。

## 3. React 内联样式

在 React 中直接在元素内使用 CSS 样式是通过内联样式来实现的。

内联样式是将 CSS 样式直接作为一个对象传递给元素的 style 属性，每个样式属性都以驼峰命名法表示，而不是传统的 CSS 属性名称。

## 4. 浏览器事件模型

### 常见事件
- **click：** 点击事件
- **contextmenu：** 右键菜单事件
- **mousemove：** 鼠标移动事件
- **keydown：** 键盘按下事件
- **scroll：** 滚动事件

### 事件监听
```javascript
element.addEventListener('event', handler, options)
```

### 事件冒泡和捕获
- **事件冒泡：** 事件从最具体的元素开始，向上传播到不太具体的元素
- **事件捕获：** 事件从不太具体的元素开始，向下传播到最具体的元素

## 5. React 框架定位

### 为什么 React 被称为 UI 库
React 官方一直自称是一个 UI 库(Library)，而不是完整的框架(Framework)。它的核心职责是构建用户界面，并没有内置路由、状态管理、数据层等全家桶功能。但在实际使用中，React 的生态非常庞大，配合 Router、Redux、Next.js 等库时，已经能承担框架的角色。

### 核心功能单一
React 只关注视图层(View)，负责把数据渲染成 UI。它不包含路由、数据管理、HTTP 请求等功能。

### 生态驱动
React 把这些功能交给社区或官方的独立库来完成，比如：
- 路由：React Router
- 状态管理：Redux、Zustand、Recoil
- 服务端渲染：Next.js、Remix

这种"拼装式"生态让 React 更灵活，但也意味着它本身不是一个完整框架。

### 对比 Vue/Angular
Vue 和 Angular 提供了更完整的"全家桶"方案（路由、状态管理、构建工具都有官方支持）。React 更像是一个"核心引擎"，需要开发者根据需求选择配套库。

### 实际使用中的"框架化"
虽然 React 自称 UI 库，但在真实项目里：
- 大型应用：几乎都会搭配 Router + 状态管理 + 构建工具，形成一个完整的框架体系
- Next.js / Remix：在 React 之上提供了路由、数据获取、SSR/SSG 等功能，使 React 生态具备了框架的能力
